## 2025-03-13 - [Hardcoded CORS Origins & Secure Configuration]\n**Vulnerability:** The Firebase Cloud Function for form submission allowed unrestricted Cross-Origin Resource Sharing (CORS) or hardcoded configurations, potentially enabling any domain to send valid requests. Furthermore, evaluating defineString parameters at the global scope would crash the Firebase CLI discovery.\n**Learning:** Implementing `cors({ origin: allowedOrigins.value() })` inside the request handler securely enforces environment-specific allowed domains and avoids the global scope evaluation crash. Avoid using `await new Promise(...)` with the cors module on OPTIONS requests as it terminates the connection before resolving the promise.\n**Prevention:** Use `defineString` to inject CORS origins dynamically during function execution and wrap logic in the `cors` middleware callback instead of a hanging Promise.
