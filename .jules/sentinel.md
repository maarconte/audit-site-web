## 2024-03-21 - [Strict validation for dynamic payloads bypassing Firestore rules]
**Vulnerability:** The `scores` object in `functions/src/index.ts` was being passed directly to `admin.firestore().collection().add()`. Since `admin.firestore()` bypasses Firestore security rules, the lack of input validation on `scores` created a vulnerability for Mass Assignment and NoSQL injection, potentially allowing arbitrary nested data, excessive keys (DoS risk), or unexpected types to be written to the database.
**Learning:** In this project, Firebase Cloud Functions use the `admin.firestore()` SDK, which runs with elevated privileges and completely bypasses `firestore.rules`. All input validation must be handled explicitly within the Cloud Functions code.
**Prevention:** Enforce strict validation for dynamic payload objects (like `scores`). Enforce flat objects, limit key counts/lengths, and validate value types before writing to Firestore. Ensure robust object checks in JavaScript by using `typeof obj === 'object' && obj !== null && !Array.isArray(obj)`.
