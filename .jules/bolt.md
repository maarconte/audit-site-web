## 2025-02-28 - [Canvas Animation Loop Anti-Pattern]
**Learning:** In Next.js/React applications, starting a `requestAnimationFrame` loop on component mount that constantly runs (even when there's nothing to draw on the canvas, like in the previous `ClickSpark` implementation) causes massive CPU and battery drain. Next.js does not magically optimize this away.
**Action:** Always start canvas animations based on user interaction (like `onClick`) and explicitly stop the loop (`cancelAnimationFrame` or exit the recursion) when the animation finishes or there are no items left to render.

## 2025-04-01 - [Hoisting Static Configurations & Identity Set Optimization]
**Learning:** In React components, mapping over a static JSON array to derive configuration data (like categories) inside `useMemo` or `useEffect` is a performance anti-pattern because it unnecessarily re-allocates objects and runs derivation logic on every render (or mount). Additionally, using `new Set(data.map((q) => q))` to deduplicate an array of objects is functionally flawed because `Set` checks strict reference equality, not structural equality. If the objects are already uniquely derived or inherently unique, the `Set` does nothing but add overhead.
**Action:** Always hoist static derivations (e.g., extracting categories from imported JSON data) outside the React component completely. This guarantees the array and its objects are created exactly once during module evaluation, preventing unnecessary allocations and avoiding redundant mapping/deduplication logic within the component lifecycle.
