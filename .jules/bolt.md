## 2025-02-28 - [Canvas Animation Loop Anti-Pattern]
**Learning:** In Next.js/React applications, starting a `requestAnimationFrame` loop on component mount that constantly runs (even when there's nothing to draw on the canvas, like in the previous `ClickSpark` implementation) causes massive CPU and battery drain. Next.js does not magically optimize this away.
**Action:** Always start canvas animations based on user interaction (like `onClick`) and explicitly stop the loop (`cancelAnimationFrame` or exit the recursion) when the animation finishes or there are no items left to render.
## 2025-02-28 - [Performance Anti-Pattern: Intermediate Object Allocation]
**Learning:** In React components (like `FormContainer`), using multiple iteration passes (e.g., mapping/reducing an array into an object, then calling `Object.values().some()` or `Object.values().reduce()`) creates unnecessary intermediate objects on every render. This increases memory allocation and garbage collection overhead, particularly when form state triggers frequent renders.
**Action:** Consolidate data transformations into single passes (e.g., using `reduce` or `.some` directly on the original array) to eliminate intermediate object creation and reduce CPU overhead.
